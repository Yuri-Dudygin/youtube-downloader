# Промт программы

Ты помогаешь развивать и поддерживать локальную Python-программу для скачивания материалов с YouTube.

## Задача программы

Программа должна принимать ссылку на YouTube-видео и сохранять связанные с ним материалы на локальный компьютер пользователя:

- `load.py` скачивает видео в лучшем доступном качестве, объединяет видео и аудио в MP4 и дополнительно создает MP3-копию аудиодорожки.
- `subs.py` скачивает субтитры без загрузки самого видео, конвертирует их в SRT, затем преобразует SRT в чистый TXT-текст без таймкодов, номеров сегментов, HTML-тегов и повторяющихся фрагментов rolling captions.

## Основной сценарий

Пользователь запускает программу через `.bat`-файлы или напрямую через Python. Ссылка может быть передана через аргумент `--url` или введена интерактивно через `--interactive`.

Пути сохранения задаются в `settings.txt`:

- `output_path` - папка для MP4-видео;
- `mp3_path` - папка для MP3-файлов;
- `subs_path` - папка для TXT-субтитров, если задана. Если `subs_path` отсутствует, используется `output_path`.

## Требования к поведению

Программа должна:

- работать на Windows в локальном окружении Python;
- использовать `yt-dlp` для получения данных с YouTube;
- использовать FFmpeg для объединения видео/аудио и конвертации субтитров;
- использовать Node.js из виртуального окружения, если он доступен, для обработки YouTube JavaScript challenge;
- использовать Chromium как обязательный браузер для авторизации в YouTube;
- сначала пробовать cookies из `cookies.txt`, затем зашифрованный кэш cookies, затем скачивание без cookies через клиенты `tv_simply` и `android_vr`, затем cookies из Chromium;
- сохранять cookies, прочитанные из Chromium, в зашифрованный DPAPI-кэш `cookies.cache.dpapi`;
- удалять и обновлять зашифрованный кэш, если YouTube отклонил сохраненные cookies;
- не скачивать плейлисты целиком, а обрабатывать только переданные видео;
- писать подробный лог последнего запуска в `last_run.log`, добавляя временную метку в начало каждой строки;
- показывать пользователю только важные сообщения, скрывая шумные предупреждения `yt-dlp`;
- продолжать обработку следующих ссылок, если одна ссылка завершилась ошибкой.

## Требования к результату

После успешного запуска `load.py` пользователь должен получить:

- MP4-файл видео в папке `output_path`;
- MP3-файл аудио в папке `mp3_path`;
- запись о ходе работы в `last_run.log`.

После успешного запуска `subs.py` пользователь должен получить:

- TXT-файл с чистым текстом субтитров в папке `subs_path` или `output_path`;
- удаленный временный SRT-файл после конвертации;
- запись о ходе работы в `last_run.log`.

## Ограничения

При изменении программы нужно сохранять простоту запуска для пользователя: `setup.bat` устанавливает зависимости, `run.bat` запускает скачивание видео, `run_subs.bat` запускает скачивание субтитров. Новые изменения не должны требовать ручной настройки, если ее можно безопасно выполнить автоматически.

Chromium является частью рабочего сценария: пользователь авторизуется в YouTube через Chromium, а программа сама читает, кэширует и обновляет cookies без ручного экспорта.
